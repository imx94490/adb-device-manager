# ADB 设备管理系统 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
一款基于 Web 技术栈构建的桌面端 ADB（Android Debug Bridge）设备管理工具，为 Android 开发者和测试人员提供直观、高效的设备管理与调试体验。

### 1.2 核心价值
- **可视化操作**：将复杂的 ADB 命令转化为直观的图形界面操作
- **设备集中管理**：统一管理多台设备的连接状态与信息
- **远程控制**：实时屏幕投射与鼠标模拟操作
- **效率提升**：拖拽安装 APK、一键抓取日志等高频操作优化

### 1.3 目标用户
- Android 应用开发者
- 测试工程师
- QA 质量保证人员
- 移动端技术支持人员

---

## 2. 功能需求

### 2.1 设备管理模块

#### 2.1.1 设备列表展示
- 展示所有已连接设备的列表
- 每台设备显示信息：
  - 设备序列号
  - 设备名称/型号
  - IP 地址（WiFi 连接设备）
  - 连接方式（USB / WiFi）
  - 在线状态（在线 / 离线 / 未授权）
  - Android 版本
  - 屏幕分辨率

#### 2.1.2 设备连接管理
- **检查设备状态**：执行 `adb devices` 刷新设备列表
- **连接设备**：通过 IP:Port 连接 WiFi 设备
  - 支持手动输入 IP 地址和端口（默认 5555）
  - 支持从历史记录中快速选择
- **断开连接**：断开指定 WiFi 设备
- **重新连接**：对离线设备一键重新连接
- **自动刷新**：可配置的自动刷新间隔

#### 2.1.3 设备信息持久化
- 本地保存已连接过的设备信息
- 支持设备备注名编辑
- 支持设备分组/标签管理

---

### 2.2 屏幕投射与远程控制模块

#### 2.2.1 屏幕实时投射
- 点击设备后打开屏幕投射窗口
- 实时显示设备屏幕画面
- 支持画面缩放适应窗口
- 支持全屏模式
- 支持屏幕旋转（跟随设备或手动切换）

#### 2.2.2 远程触控操作
- 鼠标点击模拟设备触摸点击
- 鼠标拖拽模拟滑动操作
- 支持常见按键模拟：
  - 返回键
  - Home 键
  - 菜单键/多任务
  - 音量加减
  - 电源键

#### 2.2.3 截图与录屏
- 一键截图并保存到本地
- 一键录屏（可配置时长）

---

### 2.3 APK 安装模块

#### 2.3.1 拖拽安装
- 提供专门的 APK 拖拽安装区域
- 支持将 APK 文件拖拽到设备卡片上直接安装
- 拖拽时提供视觉反馈（高亮、边框动画）

#### 2.3.2 文件选择安装
- 支持通过文件选择对话框选择 APK
- 支持历史安装记录展示

#### 2.3.3 安装过程
- 实时显示安装进度
- 安装成功/失败提示
- 失败时显示详细错误信息

---

### 2.4 日志管理模块

#### 2.4.1 日志抓取
- 执行 `adb logcat` 抓取设备日志
- 可配置日志输出路径和文件名
- 支持日志级别过滤（Verbose / Debug / Info / Warning / Error / Fatal）
- 支持 Tag 过滤
- 实时显示日志输出

#### 2.4.2 日志文件管理
- 查看已保存的日志文件列表
- 支持删除指定日志文件
- 支持打开日志文件所在目录
- 支持日志文件大小显示

#### 2.4.3 日志查看器
- 语法高亮（按级别着色）
- 支持关键词搜索
- 支持实时滚动/暂停

---

### 2.5 设置模块

#### 2.5.1 ADB 配置
- ADB 路径配置（自动检测 + 手动设置）
- ADB 连接超时设置
- 默认端口设置

#### 2.5.2 界面设置
- 主题切换（深色/浅色）
- 语言设置
- 自动刷新间隔设置

#### 2.5.3 日志配置
- 默认日志保存路径
- 日志文件命名规则
- 自动清理策略

---

## 3. 非功能需求

### 3.1 性能要求
- 应用启动时间 < 3 秒
- 设备列表刷新响应 < 1 秒
- 屏幕投射延迟 < 200ms（理想目标，受网络影响）
- 内存占用 < 500MB

### 3.2 可用性要求
- 界面响应即时反馈
- 操作错误时提供清晰的错误提示和解决方案
- 所有主要操作可通过快捷键完成

### 3.3 兼容性要求
- 支持 Windows 10/11 64 位
- 支持 Android 5.0 及以上设备
- 支持 USB 和 WiFi 连接方式

### 3.4 安全性要求
- 不收集任何用户数据
- 所有设备信息仅保存在本地
- 日志文件本地存储，不上传

---

## 4. 界面设计

### 4.1 设计风格
- **科技感深色主题**：以深色为主色调，搭配霓虹蓝/绿色强调色，营造开发者工具的专业感
- **赛博朋克美学**：融入发光边框、扫描线效果、数据流动画等元素
- **极简布局**：信息层次清晰，操作入口明确

### 4.2 核心页面

#### 4.2.1 主界面 - 设备列表
- 左侧：设备列表（卡片式布局）
- 右侧：选中设备的详细信息和快捷操作
- 顶部：工具栏（刷新按钮、添加设备按钮、设置按钮）
- 底部：状态栏（ADB 状态、设备数量）

#### 4.2.2 设备详情页
- 顶部：设备基本信息栏
- 左侧：屏幕投射区域
- 右侧：操作面板（APK安装、日志、快捷操作）
- 底部：操作日志输出区

#### 4.2.3 设置页面
- 分类标签页式布局
- 每个分类下的设置项清晰排列
- 实时保存配置

---

## 5. 技术约束

- 使用 Electron 打包为 Windows 桌面应用（.exe）
- 前端使用 React + TypeScript + Vite
- UI 框架使用 Tailwind CSS
- 调用系统 ADB 命令行工具
- 屏幕投射优先使用 scrcpy（如可用），降级使用 adb screencap

---
